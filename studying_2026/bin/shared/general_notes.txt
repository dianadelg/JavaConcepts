- Prefix vs Substring vs SubSequence -- ex: flower: 
	Prefix: the START of a string, must begin at index 0 -- "fl" "flo" "flow"
		NOT: "low" because we have to start at the beginning of the word

		NOT: "fwr" because we skipped letters inbetween
	Subsequence: characters in order, but you MAY skip -- "fwr" "flow" "fr"
		NOT: "rfe" because order changed

	This is important, it's how you understand Big O

 * BIG-O SPACE COMPLEXITY:
 * VALUES VS MEMORY
 * IMPORTANT:
 *
 * Space complexity measures:
 *
 *     how much EXTRA MEMORY is allocated
 *
 * NOT:
 *
 *     how large a VALUE becomes
 *
 *
 * Example:
 *
 *     int x = 1;
 *     x = 1000000000;
 *
 * x got MUCH larger in value,
 * BUT we still only have:
 *
 *     ONE integer variable
 *
 * Therefore:
 *
 *     O(1) space
 *
 *
 * Space grows when we allocate MORE things:
 *
 *     new int[n]
 *     HashMap
 *     ArrayList
 *     recursion stack
 *     StringBuilder growing with input
 *
 * because memory usage scales with input size.
 *
 *
 * Primitive variables are treated as:
 *
 *     constant-size memory
 *
 * Example:
 *
 *     int
 *     boolean
 *     char
 *
 * are all:
 *
 *     O(1) space
 *
 * regardless of the numeric value stored inside them.
 *
 *
 * IMPORTANT INTERVIEW TAKEAWAY:
 *
 * Ask:
 *
 *     "Am I allocating MORE memory as input grows?"
 *
 * NOT:
 *
 *     "Are the numbers themselves getting bigger?"
 *
 */
		 

		 



🚀 HIGH-YIELD DATA STRUCTURES CHEAT SHEET

========================================
ARRAY / ARRAYLIST
========================================
Use When:
- ordered data
- frequent iteration
- index access needed

Runtime:
- Access: O(1)
- Search: O(n)
- Insert/Delete middle: O(n)

Instinct:
"I need ordered items or index access."

Java:
List<String> list = new ArrayList<>();


========================================
HASHMAP
========================================
Use When:
- fast lookup
- counting frequencies
- caching
- key/value relationships

Runtime:
- Lookup/Insert/Delete average: O(1)
- Worst case: O(n)

Instinct:
"Could a hashmap avoid repeated searching?"

Common Uses:
- Two Sum
- counting chars
- grouping
- deduping

Java:
Map<String, Integer> map = new HashMap<>();


========================================
HASHSET
========================================
Use When:
- uniqueness
- "have I seen this before?"

Runtime:
- Contains/Add: O(1)

Instinct:
"I care about uniqueness/existence checks."

Java:
Set<Integer> seen = new HashSet<>();


========================================
STACK
========================================
Use When:
- nested structures
- undo behavior
- parentheses matching
- DFS

Runtime:
- Push/Pop: O(1)

Instinct:
"Last thing in should come out first."
(LIFO)

Common Problems:
- Valid Parentheses
- expression evaluation
- backtracking

Java:
Stack<Character> stack = new Stack<>();

Note:
ArrayDeque is preferred in real Java applications.


========================================
QUEUE
========================================
Use When:
- FIFO ordering
- scheduling
- BFS

Runtime:
- Add/Remove: O(1)

Instinct:
"Process things in arrival order."

Java:
Queue<Integer> queue = new LinkedList<>();


========================================
LINKED LIST
========================================
Use When:
- lots of inserts/deletes
- sequential traversal

Runtime:
- Insert/Delete at known node: O(1)
- Access/Search: O(n)

Instinct:
"I care more about modifications than random access."

Reality:
Less common in actual backend work than interviews make it seem.


========================================
TREE
========================================
Use When:
- hierarchical data
- searching/sorting
- recursive structures

Binary Search Tree Runtime:
- Average search: O(log n)
- Worst case: O(n)

Instinct:
"This data has hierarchy."


========================================
HEAP / PRIORITY QUEUE
========================================
Use When:
- always need smallest/largest item quickly
- scheduling
- top K problems

Runtime:
- Insert/Delete top: O(log n)
- Peek: O(1)

Instinct:
"I constantly need the highest/lowest priority item."

Java:
PriorityQueue<Integer> pq = new PriorityQueue<>();


========================================
TRIE
========================================
Use When:
- autocomplete
- prefix searching
- dictionaries

Runtime:
- Insert/Search: O(m)
  (m = word length)

Instinct:
"I care about prefixes."

Concept:
Stores words character-by-character for efficient prefix lookup.






		 
Mental Decision Tree
	Need fast lookup? 👉 HashMap
	
	Need uniqueness?👉 HashSet
	
	Need ordering + indexes?👉 ArrayList
	
	Need LIFO?👉 Stack
	
	Need FIFO?👉 Queue
	
	Need min/max repeatedly?👉 PriorityQueue
	
	Need hierarchy?👉 Tree
